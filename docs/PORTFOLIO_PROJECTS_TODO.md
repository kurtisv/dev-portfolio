# Portfolio Projects

## Objective

Maintain a recruiter-ready collection of small full-stack portfolio applications. Each project should have a live URL, public repository when appropriate, a clear case study, and a short explanation of the engineering choices.

## Published projects

- [x] SupportDesk Lite: support ticket workflow and dashboard.
- [x] QuotePilot: quote intake and business document workflow.
- [x] Luma Studio: interior design studio portfolio.
- [x] API Meter: API usage, keys, plans, and documentation.

## Next project standards

- [ ] Create a separate GitHub repository.
- [ ] Keep the repository public when the project is meant for recruiting.
- [ ] Build from a clean foundation, not inside another project repository.
- [ ] Add FR/EN when the project is portfolio-facing.
- [ ] Add live deployment URL to `apps/web/src/data/portfolio.ts`.
- [ ] Run lint, typecheck, tests, and production build before deploy.
- [ ] Update the Symphonee project note after deploy.

## Review notes

- The developer profile site is the central index for the portfolio collection.
- Keep project descriptions concrete: problem, build, visible product, and recruiter value.
- Avoid placeholder copy or internal setup language on public pages.
